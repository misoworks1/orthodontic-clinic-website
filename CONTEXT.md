## 2. 개발 환경

### 기술 스택
```
HTML5 + CSS3 + Vanilla JavaScript
```

### 선택 근거 (레퍼런스 분석 기반)
- 한국 교정치과 홈페이지의 90%가 jQuery 기반 → **Vanilla JS로 대체** (경량화)
- 프레임워크(React/Vue) 불필요: 정적 콘텐츠 중심, 서버사이드 로직 최소
- 단일 HTML 파일 또는 소수 HTML 파일 구조 → **커스텀 CSS** 사용 (Tailwind 불필요)
- 외부 라이브러리는 CDN으로 최소한만 사용

### 사용 라이브러리 (CDN)
| 라이브러리 | 버전 | 용도 | 근거 |
|-----------|------|------|------|
| Swiper.js | 11.x | 메인 슬라이더, B/A 갤러리 | 레퍼런스 50%+ 사용, 가장 보편적 |
| AOS | 2.x | 스크롤 애니메이션 | 레퍼런스 30%+ 사용, 경량 |
| - | - | 네비게이션 | 순수 CSS + JS로 구현 |

### 폰트
```
본문: Pretendard (한글) + Pretendard (영문)
강조/제목: 선택적으로 세리프체 혼합 가능
아이콘: 직접 SVG 또는 Material Icons CDN
```

### 개발 도구
```
에디터: VS Code
버전관리: Git + GitHub
배포: GitHub Pages 또는 Netlify (추후 결정)
이미지 최적화: WebP 포맷 권장
```

## 4. 페이지 구성 (정보 아키텍처)

### 메인 페이지 섹션 구조
레퍼런스 분석에서 도출한 교정치과 홈페이지 필수/권장 섹션:

```
[Header] 고정 네비게이션 + 로고 + 예약 CTA
    ↓
[Hero] 풀스크린 이미지/비디오 슬라이더 + 핵심 메시지
    ↓
[About] 병원 소개 + 차별화 포인트
    ↓
[Doctor] 원장/의료진 프로필 + 학력/경력
    ↓
[Treatment] 교정 진료 종류 (인비절라인, 세라믹, 설측 등)
    ↓
[Cases] Before/After 갤러리 (증상별 필터링)
    ↓
[Reviews] 환자 후기/리뷰
    ↓
[Facility] 시설 사진 갤러리
    ↓
[Location] 오시는 길 + 네이버 지도 + 진료시간
    ↓
[Footer] 연락처 + SNS + 사업자 정보
    ↓
[Quick Menu] 우측/하단 고정 (전화, 카톡, 네이버예약)
```

## 5. 반응형 디자인 전략

### 브레이크포인트
```css
/* 모바일 퍼스트 */
@media (min-width: 768px)  { /* 태블릿 */ }
@media (min-width: 1024px) { /* 데스크톱 */ }
@media (min-width: 1280px) { /* 와이드 데스크톱 */ }
```

### 네비게이션 반응형
- **1024px 이상**: 수평 드롭다운 메뉴
- **1024px 미만**: 햄버거 메뉴 → 슬라이드 사이드바

### 히어로 반응형
- **데스크톱**: 풀스크린 100vh, 대형 타이포
- **모바일**: 60~70vh, 축소된 폰트 크기, 별도 모바일 이미지 제공

## 6. 핵심 컴포넌트 설계

### 6-1. 고정 헤더
```
- 스크롤 시 배경색 변화 (투명 → 불투명)
- 로고 좌측, 메뉴 우측 배치
- 예약 CTA 버튼 항상 노출
- 스크롤 방향 감지 (선택): 아래 스크롤 시 숨김, 위 스크롤 시 표시
```

### 6-2. 히어로 슬라이더
```
- Swiper.js 기반
- 자동재생 (4~6초 간격)
- 페이드 또는 슬라이드 전환
- 페이지네이션 도트
- 모바일 터치 스와이프 지원
```

### 6-3. Before/After 갤러리
```
- Swiper.js 기반 캐러셀
- 증상별 필터 탭 (돌출입, 덧니, 벌어진치아, 주걱턱 등)
- 이미지 레이지 로딩
- 클릭 시 모달 확대 뷰
```

### 6-4. 고정 퀵메뉴
```
- 우측 또는 하단 고정 위치
- 전화 바로걸기
- 카카오톡 상담
- 네이버 예약
- 모바일에서는 하단 바 형태
```

### 6-5. 온라인 상담 폼
```
- 이름, 연락처, 진료 분야 선택, 문의 내용
- 입력값 유효성 검사 (전화번호 정규식)
- 이중 제출 방지
- 성공/실패 피드백 표시
```

## 7. 성능 최적화 전략

### 이미지
- WebP 포맷 우선, fallback으로 JPG
- 히어로 이미지: 데스크톱/모바일 별도 제공 (`<picture>` 태그)
- 갤러리 이미지: `loading="lazy"` 적용
- 적정 크기: 히어로 1920x1080, 갤러리 800x600

### CSS/JS
- 커스텀 CSS → 미니파이 (배포 시)
- 외부 라이브러리 CDN 로드
- `defer` / `async` 스크립트 로딩
- Critical CSS 인라인 삽입 (선택)

### SEO
- 시맨틱 HTML5 태그 (`<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`)
- `<meta>` 태그: title, description, og:image 등
- 구조화 데이터 (Schema.org - LocalBusiness, Dentist)
- 네이버 서치어드바이저 메타태그

## 10. 주의사항

### 의료법 관련
- 치료 전후 사진 게시 시 환자 동의 필요 (의료법 제56조)
- 과대광고 금지: "최고", "최초" 등 표현 주의
- 치료 비용 표시 시 의료광고 심의 기준 준수

### 접근성
- 텍스트 대비 비율 4.5:1 이상 (WCAG AA)
- 키보드 네비게이션 지원
- alt 텍스트 필수
- focus 표시 유지

### 브라우저 지원
- Chrome 90+, Safari 15+, Edge 90+, Firefox 90+
- IE 미지원
