## 3. 프론트엔드 디자인 패턴별 분류

### 패턴 A: 풀스크린 비디오 히어로형

**해당 사이트**: E01 Holy City, E04 Ballard, E10 Serrano, E12 Rucker, E13 Floyd Family, E17 Appel, K06 뉴엔치과

**프론트엔드 기술적 특징**:
- `<video>` 태그 autoplay, muted, loop 속성으로 배경 비디오 구현
- object-fit: cover로 풀스크린 비율 유지
- 비디오 위 텍스트 오버레이 (z-index 레이어링)
- 모바일에서는 정적 이미지 fallback으로 성능 최적화
- 일반적으로 60~100vh 높이 설정
- 스크롤 인디케이터 (바운스 애니메이션 화살표)

**장점**: 즉각적 시각 임팩트, 클리닉 분위기 직관적 전달
**주의**: 모바일 데이터 소비, 로딩 속도

---

### 패턴 B: 이미지 슬라이더/캐러셀 히어로형

**해당 사이트**: K01 서울클리어, K02 연세굿데이, K03 스마일디, K04 아이디치과, K08 서울특별, K10 예바치과, E02 Dunn, E31 Devlin

**프론트엔드 기술적 특징**:
- Swiper.js 또는 Slick.js 슬라이더 라이브러리 활용
- 페이드(fade) 또는 슬라이드(slide) 전환 효과
- 자동 재생 (2~5초 간격), 점/화살표 네비게이션
- 터치 스와이프 제스처 지원 (모바일)
- CSS transition/transform으로 부드러운 전환
- 불릿 페이지네이션 또는 썸네일 미리보기

**장점**: 다양한 메시지/프로모션 동시 노출, 콘텐츠 회전
**주의**: 사용자가 모든 슬라이드를 보지 않을 수 있음

### 패턴 C: 스크롤 스토리텔링형 (Scroll-Triggered Animation)

**해당 사이트**: K01 서울클리어(GSAP), K02 연세굿데이(AOS), K07 연세올곧은(View Transitions), E10 Serrano, E36 London Orthodontic, E37 Six B Dentaire

**프론트엔드 기술적 특징**:
- GSAP (GreenSock) + ScrollTrigger로 정밀한 스크롤 연동 애니메이션
- AOS (Animate On Scroll) 라이브러리로 요소별 페이드인/슬라이드인
- Intersection Observer API 기반 뷰포트 감지
- translate3d, opacity, scale 트랜스폼 조합
- 패럴랙스 효과 (배경/전경 속도 차이)
- 키네틱 타이포그래피 (스크롤에 따라 텍스트 변형)
- cubic-bezier 이징으로 자연스러운 가감속

**장점**: 몰입감 높은 사용자 경험, 정보의 순차적 공개
**주의**: 과도한 애니메이션은 성능 저하/접근성 문제

### 패턴 D: 미니멀리즘형 (Clean & Minimal)

**프론트엔드 기술적 특징**:

- 제한된 컬러 팔레트 (2~3색)
- 시스템 폰트 또는 얇은 산세리프 서체
- 그리드 기반 카드 레이아웃
- 최소한의 애니메이션 (호버 효과 정도)

- 큰 여백과 고해상도 사진 조합

**장점**: 빠른 로딩, 높은 가독성, 시대 초월적 디자인
**주의**: 차별화 어려움, 밋밋해 보일 수 있음

- 브레이크포인트별 columns 조정 (모바일 1열, 태블릿 2열, 데스크톱 3~4열)
